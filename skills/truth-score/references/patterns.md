# 44 个中文 AI 味 Pattern

这些 pattern 是 Article To Truth 的核心规则库。评测、改写和质检时必须引用编号。单个 pattern 命中不等于 AI 写作，判断时看组合、密度和上下文。

## Content Patterns

### C01 意义膨胀

**信号**：标志着、意味着、开启了、关键转折、重要里程碑、深远影响。

**问题**：把普通功能、普通事件写成行业级变化。

**改法**：写具体发生了什么、影响谁、限制是什么。

Before:

```text
这次更新标志着内容创作进入全新阶段。
```

After:

```text
这次更新增加了批量改标题和按语气重写两项功能。
```

### C02 宏大背景铺垫

**信号**：在当今时代、随着技术发展、在数字化浪潮下、面对复杂环境。

**问题**：开头先搭一个大舞台，但没有提供有效信息。

**改法**：直接进入具体问题。

Before:

```text
在当今快速发展的数字时代，企业越来越重视内容效率。
```

After:

```text
很多团队的问题不是不会写，而是同一篇稿子要改出公众号、小红书和官网三个版本。
```

### C03 模糊归因

**信号**：行业报告显示、专家认为、业内人士指出、多数用户反馈。

**问题**：借模糊权威增强可信度。

**改法**：补明确来源；没有来源就删掉或改成作者判断。

Before:

```text
多位专家认为，这类工具会持续改变内容行业。
```

After:

```text
如果没有具体来源，这句话不该写进正文。可以改成：我倾向于认为这类工具会先影响高频改稿场景。
```

### C04 伪数据和伪案例

**信号**：大量用户、显著提升、普遍反馈、真实案例证明，但没有样本和来源。

**问题**：看起来具体，其实不可核验。

**改法**：删除数字感表达，或标成假设、观察、待验证。

Before:

```text
大量用户反馈，使用后效率显著提升。
```

After:

```text
如果你的工作是反复改标题和摘要，它可能会省时间。但具体能省多少，要看素材质量和审核流程。
```

### C05 万能价值判断

**信号**：提升效率、优化体验、降低成本、增强竞争力、创造价值。

**问题**：说了好处，没有说机制。

**改法**：补“通过什么动作影响什么环节”。

Before:

```text
该方案能够提升团队协作效率。
```

After:

```text
它把选题、初稿、审稿意见放在同一个文档里，减少来回找聊天记录的时间。
```

### C06 缺少适用边界

**信号**：适用于所有人、全面覆盖、无论什么场景、都可以。

**问题**：把有条件的建议写成普遍结论。

**改法**：写适合谁、不适合谁、前提条件。

Before:

```text
每个内容团队都应该引入这套流程。
```

After:

```text
如果团队每周要产出多平台内容，这套流程值得试。偶尔写一篇文章的人，可能用不上这么重的步骤。
```

### C07 伪个人经验

**信号**：我亲测、我身边很多人、真实经历告诉我，但内容像通用话术。

**问题**：用“我”包装不存在或不可验证的经验。

**改法**：没有真实经历就不要写成亲历；改为可验证观察或条件判断。

Before:

```text
我身边很多创作者用了之后都说离不开它。
```

After:

```text
我更愿意把它看成初稿工具。它能不能进入日常流程，取决于后面的人工审稿是否顺手。
```

### C08 通用乐观结尾

**信号**：未来可期、让我们拭目以待、无限可能、开启新篇章。

**问题**：结尾脱离正文事实，强行升华。

**改法**：落到下一步、风险或具体判断。

Before:

```text
未来，AI 内容创作必将迎来更加广阔的发展空间。
```

After:

```text
真正要观察的是发布前那一步：它能不能帮作者发现空话，而不只是把空话改得更顺。
```

### C09 作文式通用细节

**信号**：烤红薯、雨伞、路灯、一碗面、旧街口、不吃香菜这类细节出现得很具体，但换到任何人、任何关系里也成立。

**问题**：看似有生活感，实际是为了证明主题而摆放的安全细节，没有真实关系里的偏差、别扭、时间、地点或不可替代性。

**改法**：保留能说明关系的细节，删掉只负责“显得温暖”的细节；如果没有真实素材，就把细节写得更有限、更局部，不要伪造亲历。

Before:

```text
友情藏在一把递过来的伞里，也藏在多年后见面仍然能自然坐下的一碗面里。
```

After:

```text
我记得的是一次很小的事。她把伞递过来时没看我，只说“你别又感冒”。那句话不漂亮，但我到现在还记得。
```

### C10 细节平均分配

**信号**：每个段落都刚好有一个细节、一个感受、一个小结；细节都准确服务主题，没有旁枝、闲话、误听、局部视角或不完全解释的东西。

**问题**：人写回忆或作文时，常会留下不那么“有用”的材料：饭桌上听来的话、路过时瞥见的动作、一句方言、一个没解释完的人际关系。AI 润色常把这些毛边删掉，只留下对主题有用的均衡素材。

**改法**：不要为了完整而把素材排成整齐证据链。保留少量能显示叙述位置的旁枝细节，但不要编造新的亲历或假装知道更多。

Before:

```text
她总是用自己的方式照顾生活，也用这份从容影响着身边的人。
```

After:

```text
我听爷爷在饭桌上说过一句：“她不急。”至于她到底急不急，我那时看不出来，只记得她常在院里剪那盆歪掉的盆栽。
```

## Language Patterns

### L01 机械连接词堆叠

**信号**：此外、然而、值得注意的是、总的来说、与此同时、换句话说。

**问题**：逻辑靠连接词硬接，不靠内容自然推进。

**改法**：能删就删；需要转折时写出真实冲突。

### L02 虚浮动词

**信号**：赋能、助力、彰显、释放、打造、沉淀、构建、驱动。

**问题**：动词很热闹，但动作不清楚。

**改法**：换成具体行为：整理、筛选、合并、比较、生成、检查、提醒。

### L03 过度形容

**信号**：高效、强大、智能、便捷、全面、丰富、深刻、无缝、先进。

**问题**：用形容词替代证据。

**改法**：写结果、过程或限制。

### L04 名词化堆叠

**信号**：能力建设、效率提升、流程优化、价值释放、生态构建。

**问题**：句子抽象到看不见人和动作。

**改法**：恢复主语和动词。

Before:

```text
通过流程优化实现内容效率提升。
```

After:

```text
先把选题、初稿和审稿拆开处理，编辑不用每次从头找材料。
```

### L05 发布会腔

**信号**：全新升级、重磅发布、焕新体验、强势来袭、领先方案。

**问题**：像广告稿，不像真实说明。

**改法**：删掉姿态词，写变了什么。

### L06 行业黑话连发

**信号**：闭环、链路、矩阵、抓手、颗粒度、心智、飞轮、方法论。

**问题**：用熟词制造专业感。

**改法**：保留业务必要词，删掉装饰词。

### L07 绝对化表达

**信号**：彻底、一定、完全、所有、最佳、唯一、必然。

**问题**：把条件性判断写得过满。

**改法**：加入条件、范围或证据。

### L08 口水化伪人味

**信号**：说白了、真的很香、狠狠爱了、懂的都懂、狠狠拿捏。

**问题**：为了不像 AI，变成另一种模板。

**改法**：保留自然语气，但让判断具体。

### L09 教学解释腔

**信号**：友情大概就是这样、人越长大越知道、真正重要的是、我们要明白、这告诉我们。

**问题**：把散文、随笔、故事写成概念讲解或作文总结，像在向读者解释主题。

**改法**：少下定义，多留在场景、动作和未说透的判断里；需要总结时，也让总结带有作者个人的偏向。

Before:

```text
友情大概就是这样，它不总是热烈，却总在关键时刻给人力量。
```

After:

```text
后来我很少再说“最好的朋友”这种话。说出来太满，也不太像我们。
```

### L10 词汇过度统一

**信号**：整篇反复使用同一组温柔、安全、珍贵、普通、自然、具体、安静等抽象词，语气稳定到没有个人偏差。

**问题**：词汇过于一致会让文本像被统一润色过，缺少作者临场选择词语时的轻微不均匀。

**改法**：保留关键词的必要重复，但让不同段落承担不同质感；可以出现更具体、更不圆润的词，不必全篇维持同一种温柔语调。

### L11 过度中性语气

**信号**：整篇语气始终客观、稳妥、温和，不生气、不迟疑、不偏心，也没有作者自己的小判断。

**问题**：过于中性的表达会抹掉人的情绪波动。真实作者常会有偏爱、误解、犹豫、突然的肯定或一句不完全讲理的话。

**改法**：根据文体保留合理的情绪温差。散文和故事可以留下偏心、疑惑、轻微别扭；产品和报告则保持克制，但允许作者给出有边界的判断。

### L12 精修无毛边

**信号**：句子干净到没有草稿痕迹，所有词都准确、顺滑、漂亮，没有重复、口头残留、半截判断或不那么高级但贴近场景的词。

**问题**：过度精修会把文本洗成统一润色风。检测器常把“太干净、太会写、太没有临场选择痕迹”视为 AI 特征。

**改法**：不要故意制造错别字或病句；可以保留有意义的重复、普通词、口头转折和局部不圆满，让文章像一个人在现场回忆或判断。

## Structure Patterns

### S01 不仅而且结构

**信号**：不仅……而且……、不仅是……更是……。

**问题**：常用来制造层次，实际只是升格。

**改法**：直接写两个事实，或保留真正有递进关系的一项。

### S02 不是而是结构

**信号**：不是……而是……、与其说……不如说……。

**问题**：制造洞察感，但经常只是换说法。

**改法**：直接陈述判断。

### S03 三件套

**信号**：效率、质量、体验；快速、稳定、强大；选题、创作、分发。

**问题**：强行凑三项显得完整。

**改法**：有几项写几项，按重要性排序。

### S04 同义词轮换

**信号**：同一个对象被反复换叫法，例如工具、平台、系统、方案。

**问题**：为了避免重复造成理解负担。

**改法**：用最清楚的词重复。

### S05 伪金句

**信号**：X 是 Y 的 Z、不是工具而是镜子、真正的价值在于。

**问题**：句子像海报，但没有增加信息。

**改法**：把比喻还原成具体判断。

### S06 短句戏剧化

**信号**：连续短句制造顿挫：没有噪音。没有等待。没有妥协。

**问题**：节奏像宣传片旁白。

**改法**：合并为自然句，保留必要强调。

### S07 段落同模

**信号**：每段都是“先定义，再好处，再总结”。

**问题**：读起来像自动生成的大纲展开。

**改法**：让段落承担不同功能：事实、例子、判断、边界、反例。

### S08 过度结构化推理

**信号**：文章按“提出结论 -> 解释原因 -> 举例证明 -> 总结升华”稳定推进，每段都像在完成一个清晰论点。

**问题**：真实随笔、故事和情感文章不总按标准推理链条前进。过度结构化会让文本像自动生成的大纲展开。

**改法**：允许局部跳开、插入不完全解释的细节、把判断放晚一点，或让某段只承担一个片段而不是完整论证。

### S09 过顺低跳跃

**信号**：句子异常顺滑，段落衔接没有停顿、迟疑、岔开或信息落差；读者几乎能预测下一句会怎样收束。

**问题**：过低困惑度会让文本显得被模型抚平。人类写作常有轻微跳跃、没说满的地方和不完全对称的段落长度。

**改法**：不要故意写乱，而是在不损伤理解的前提下保留一点不均匀：短段、半句判断、未解释透的细节、突然回到具体场景。

### S10 标准温情隐喻

**信号**：友情像旧外套、像路灯、像一盏灯、像港湾、像一碗热面、像冬天里的火。

**问题**：这些隐喻安全、顺手、温暖，但过于常见，容易成为 AI 生成的情感结尾模板。

**改法**：能不用比喻就不用；必须用时，让比喻来自本文独有的物件、动作或关系，而不是通用温情素材。

### S11 无旁枝叙事

**信号**：文章一路直奔主题，所有段落都按同一方向推进，没有偶然听来的话、无关紧要的人、岔开的动作、局部误解或后来才想明白的部分。

**问题**：过度连贯会让文本像线性生成。人类叙事常有绕路：先看见动作，再听见闲话，后来才把它和主题勉强连上，甚至有些信息不完全服务中心。

**改法**：在不伤害清晰度的前提下，保留少量自然旁枝。旁枝应该来自原文素材或用户补充，不能凭空加“真实生活感”。

### S12 可预测句法

**信号**：大量句子使用相似结构，例如“它不是……而是……”“有时……有时……”“不是……而是被……记得”“这句话很普通，却……”，读者能提前猜到下一句的收束方式。

**问题**：句式过于稳定会制造模板感。即使单句都通顺，整篇也会显得像套了一个节奏模板。

**改法**：改变句子功能和长度：有的句子只写动作，有的只写判断，有的保留短停顿；不要每一句都负责递进、对照或升华。

## Format Patterns

### F01 粗体冒号列表

**信号**：`**效率：**...`、`**体验：**...`、`**安全：**...`。

**问题**：像模型默认答案，不像自然文章。

**改法**：能合并成段落就合并；需要列表时让每项信息不同。

### F02 过度标题化

**信号**：小标题密集，每个标题下面只有一两句。

**问题**：结构抢过内容。

**改法**：删掉重复标题，保留真正帮助扫读的层级。

### F03 标题复读

**信号**：标题后第一句只是重复标题。

**问题**：浪费读者注意力。

**改法**：标题后直接进入事实或例子。

### F04 符号装饰

**信号**：emoji、过多感叹号、箭头、星标、装饰性分隔。

**问题**：强化模板感，尤其在正式文章里。

**改法**：删除装饰；保留业务必要符号。

## Bot Patterns

### B01 对话残留

**信号**：当然可以、好问题、希望这对你有帮助、如果你想让我继续。

**问题**：聊天回复被粘进正文。

**改法**：删除，正文直接开始。

### B02 知识截止和能力免责声明

**信号**：截至我的知识更新、基于可用信息、我无法访问实时数据。

**问题**：模型限制进入成稿。

**改法**：查证后写具体来源；无法查证就删掉或标为待核对。

### B03 过度顺从

**信号**：你说得完全正确、非常棒的问题、我完全理解你的需求。

**问题**：客服式讨好，不属于正文。

**改法**：直接回应内容。

## Protect Patterns

### P01 不要误伤正常正式写作

正式、准确、语法完整不等于 AI。政府公告、合同说明、技术文档、百科条目本来就可能中性克制。不要强行加入第一人称、吐槽或网感。

### P02 保留真实的人类痕迹

遇到这些内容时优先保留：具体细节、真实犹豫、可解释的个人判断、时代或圈层相关表达、作者习惯性句式、必要的重复。过度清洗会把人味洗掉。

### P03 不把毛边误判成低质

轻微重复、普通词、局部视角、口语句、方言、非核心旁枝、没有完全解释透的关系，不一定是问题。它们可能正是人工写作的来源感。

保护边界：不要为了“像人写”故意加错别字、病句、方言或混乱跳跃；只保留原文已有的有效毛边，或在虚构创作中让毛边服务人物、场景和叙述视角。
