# Truth Score

Truth Score 是一个独立评分 Skill，只负责对已有中文文本进行评测和问题定位。

## Core Contract

- 保留原文事实、文体、范围和责任边界，不把不同文体强行套入同一标准。
- 不以规避检测器为目标；评分衡量的是具体性、可信度、自然度和模板化风险。
- 每个关键判断都给出 pattern 编号和证据句，不只凭词频或单个连接词下结论。
- 不把正式写作、专业术语、必要列表、克制表达或轻微不均匀单独判成 AI 味。
- 只评测和定位问题，不输出完整改写稿。

## Workflow

1. 判断文本类型：文章、公众号、产品介绍、营销文案、小红书、短视频脚本、邮件、报告等。
2. 锁定事实和文体，不把正式、清楚、列表化本身误判为 AI 味。
3. 从 pattern 中选择最关键的命中项。短文本列 1-3 个，长文本列 3-8 个。
4. 每个命中项给出证据句，不只列词。
5. 对散文、随笔、故事、作文和情感类文本，额外检查是否有作文式通用细节、细节平均分配、教学解释腔、过度中性、精修无毛边、过度结构化、过顺低跳跃、标准温情隐喻、无旁枝叙事和可预测句法。
6. 同时做误判保护：局部视角、旁枝细节、必要重复、普通词、方言口气和轻微不均匀，不要单独判成低质。
7. 按 `rubric.md` 给出 100 分制真实感评分和对应的 AI 味风险等级。
8. 给出修改优先级：先事实可信度，再具体性和结构，最后语言润色。

## Output

默认使用这个结构：

```text
真实感评分：xx/100（越高越自然）
AI 味风险：低 / 较低 / 中 / 高 / 很高

命中 pattern：
1. [C03] ...：证据句“...”
2. [L02] ...：证据句“...”
3. [S03] ...：证据句“...”

扣分说明：
- 事实和可信度：...
- 具体性和信息密度：...
- 语言自然度：...

优先修改：
1. ...
2. ...
3. ...
```

不要附完整改写稿，也不要主动把任务延伸为改写。

## Boundaries

- 不以规避检测器为目标。
- 不把正常正式写作、专业术语、必要列表或克制表达单独判成 AI 味。
- 文本涉及医疗、法律、金融等高风险建议时，重点检查来源、边界和确定性表达。
- 用户同时要求评分和改写时，交由 `article-to-truth` 处理完整复合流程。
