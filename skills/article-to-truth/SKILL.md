# Article To Truth

Article To Truth 的目标是把文章带回人类表达的真实与自然。使用本技能时，把 AI 初稿视为素材和结构草稿，而不是可直接发布的成品。

## Intent Modes

根据用户的最终目标选择一个模式；请求同时包含多个目标时，合并为对应的复合模式。不要要求用户记忆模式名或额外命令。

- **Generation**：从零创作文章、小说、故事、公众号、营销文案、脚本、邮件或报告。
- **Evaluation**：只评测已有文本，输出真实感评分、AI 味风险、pattern 证据和修改优先级，默认不输出完整改写稿。
- **Rewrite**：只改写已有文本，保留事实、范围、责任边界和作者意图，默认不展开完整评分报告。
- **Evaluation And Rewrite**：先评测原文，再改写，最后复评终稿。
- **Voice Calibration**：分析作者样稿的稳定特征，再据此创作或改写。
- **Rules And Review**：制定写作规则、SOP、验收标准，或执行综合审稿。

## Core Contract

- 保留事实、范围、责任边界和原文意图。
- 不编造数据、来源、采访、案例、用户反馈、个人经历。
- 不以规避检测器为目标；目标是提升真实度、具体性、可信度和可读性。
- 不把所有文本改成同一种“松弛口语”。技术文、报告、公告、营销稿、个人随笔应保留各自的合理语气。
- 评测只输出一个数值体系：`真实感评分`，分数越高越自然。AI 味只用“低 / 较低 / 中 / 高 / 很高”表达，不再输出方向相反的“AI 味评分”或百分比。
- pattern 是内部分析工具。评测时展示关键 pattern；改写时只在用户需要说明时用 pattern 解释主要改动；原创生成默认不展示自审过程。
- 从零生成文章、诗歌、小说、故事或广告创意时，先把本技能当成写作准则：确定文体、声音、素材密度和禁止 pattern，再生成正文。
- 虚构创作可以创造人物、情节和意象；非虚构内容不能伪造数据、来源、采访、案例、用户反馈或亲身经历。

## Pattern System

本技能使用 44 个中文原生 AI 味 pattern。pattern 编号稳定，不要随意改名或重排。

- `C` Content：内容事实类，处理空泛、伪来源、伪经验、边界缺失、作文式通用细节和细节平均分配。
- `L` Language：语言词汇类，处理套话、虚浮动词、名词化、过度形容、教学解释腔、词汇过度统一、过度中性和精修无毛边。
- `S` Structure：句式结构类，处理机械转折、三段式、同义词轮换、伪金句、过度结构化、过顺低跳跃、标准温情隐喻、无旁枝叙事和可预测句法。
- `F` Format：格式排版类，处理粗体冒号列表、标题化和符号装饰。
- `B` Bot：对话残留类，处理机器人口吻、知识截止和过度顺从。
- `P` Protect：误判保护类，避免把正常人类写作误伤。

评测时引用 3-8 个最关键的 pattern 编号；短文本可少于 3 个。改写任务只在改动说明中引用必要的 pattern，不为凑数量重复解释。

## Rewrite Loop

执行改写时在内部按这个顺序：

1. **锁定事实**：标出不能改动的主体、数字、时间、来源、承诺范围。
2. **识别 pattern**：列出最明显的 pattern 编号和证据句。
3. **初改**：删除空话，补具体场景，打散模板结构，保留原文覆盖范围。
4. **复审**：问自己“这版哪里还明显像 AI 写的？”列 2-5 条。
5. **终改**：解决复审问题，输出最终版本。
6. **按需说明改动**：用户要求说明，或任务涉及事实边界时，用 pattern 编号解释最重要的改动。

默认只输出终稿和简短改动说明，不展示初改版或内部复审。用户说“只要正文”“只输出终稿”时只给终稿；只有用户明确要求过程、诊断或完整审稿记录时，才展示 pattern 证据与复审过程。

## Generation Loop

执行从零生成时按这个顺序：

1. **识别文体**：散文诗、小说、故事、公众号、营销文案、报告等，不同文体使用不同语气。
2. **建立素材底盘**：从用户提示中提取场景、人物、意象、观点、受众和限制；信息不足时做合理创作选择，不要写成万能模板。
3. **选择声音**：确定叙述距离、情绪温度、句长、节奏和是否使用第一人称。
4. **避开高风险 pattern**：生成前重点避开 `C01`, `C02`, `C05`, `C08`, `C09`, `C10`, `L02`, `L03`, `L09`, `L11`, `L12`, `S03`, `S05`, `S08`, `S09`, `S10`, `S11`, `S12`, `F02`。
5. **生成正文**：优先写具体画面、动作、判断和细节，不先写宏大价值。
6. **内部自审**：用 pattern 做一遍快速复审，删掉模板句和空泛升华。
7. **交付**：用户只要正文时，默认只给正文；用户要求说明时，再附简短创作说明或自审结果。

## Truth Rules

改写时优先做这些事：

- 把抽象价值改成具体动作、对象、场景和边界。
- 把模糊归因改成明确来源；没有来源就删掉或改成作者判断。
- 把确定结论改成有条件的判断，除非原文有证据支持。
- 把宣传腔改成事实、限制和使用建议。
- 把机械结构改成符合内容推进的段落。
- 把作文式通用细节改成更具体、更局部、更不可替代的关系痕迹；没有真实素材时不要伪造亲历。
- 把过于完整、过于顺滑的满分作文结构打散成更符合文体的推进方式。
- 把过度干净、中性和可预测的句子改成更有叙述位置的表达：谁看见的、在哪里听来的、作者当时知道多少。
- 保留原文已有的有效毛边：必要重复、普通词、旁枝细节、局部视角、轻微迟疑；不要把它们全部润平。

不要做这些事：

- 不要为了“人味”加入不存在的亲身经历。
- 不要为了“自然”把正式文本改得油腻、口水化。
- 不要为了“真实”故意写错、写乱或制造无意义跳跃。
- 不要删掉原文重要信息，只因为它读起来像 AI。
- 不要把所有连接词都删光，导致逻辑断裂。

```text
1. ...

3. ...

1. ...

```

不要再附“AI 味评分：xx/100”“AI 生成概率”或其他方向相反的数值。

### Rewrite

```text

...

```

用户明确要求“只输出终稿”时，省略标题之外的所有说明；如果上下文已经清楚，标题也可以省略。


## Hard Stops

遇到以下情况，先说明风险，再继续做安全部分：

- 用户要求伪造真人经历、用户反馈、实验结果、采访或来源。
- 用户要求以欺骗学校、雇主、平台或检测系统为主要目的。
- 文本涉及医疗、法律、金融等高风险建议，但缺少来源和限定条件。
- 改写会改变事实、承诺范围、责任边界或利益相关方。

在这些情况下，把任务改成“提升清晰度、真实性和合规表达”。
