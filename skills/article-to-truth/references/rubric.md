# 真实感评分与 AI 味风险量表

使用 100 分制的真实感评分，分数越高表示文本越自然、具体、可信、可发布。它是唯一的数值分数；不要同时输出方向相反的“AI 味评分”、AI 概率或检测器概率。AI 味只给风险等级、pattern 编号和证据句，不要只说“AI 味重”。

## 权重

| 维度 | 分值 | 主要关联 pattern |
|---|---:|---|
| 事实和可信度 | 25 | C03, C04, C06, C07, B02 |
| 具体性和信息密度 | 20 | C01, C02, C05, C09, C10, L04 |
| 语言自然度 | 20 | L01-L12, S09, S12 |
| 结构节奏 | 15 | S01-S12, F01-F03 |
| 作者声音 | 10 | C07, C09, C10, L08, L09, L10, L11, L12, P02, P03 |
| 格式洁净度 | 5 | F01-F04, B01 |
| 误判控制 | 5 | P01, P02, P03 |

## 分数与风险解释

- 90-100，AI 味风险低：可以发布。只有少量局部润色空间。
- 80-89，AI 味风险较低：基本可用。存在局部模板感，但不影响整体可信度。
- 70-79，AI 味风险中：需要改。常见套话、结构工整或边界不足明显。
- 60-69，AI 味风险高：需要重写。多类 pattern 成簇出现。
- 60 以下，AI 味风险很高：建议回到素材层重做。文本更像自动生成稿，而不是可编辑初稿。

## 校准锚点

先找最接近的锚点，再按维度微调。不要因为命中某个单词就把分数拉到另一个区间。

- **95 分**：事实、边界和叙述位置都清楚，表达自然且有具体信息；只剩不影响发布的局部问题。
- **85 分**：整体可信、可用，存在少量模板句或节奏问题，但没有成簇出现。
- **75 分**：有真实信息底盘，但多处套话、结构工整或作者声音不足，需要一次有针对性的编辑。
- **65 分**：模板结构和虚浮表达主导文本，具体信息被遮住，需要重写主要段落。
- **50 分**：空泛、伪来源、伪经历、事实边界改变或无依据的高风险结论明显，必须回到素材层处理。

## 扣分方法

每个维度从该维度满分开始，根据证据扣分，最后相加得到总分。pattern 用来解释证据，不是额外的一套扣分表。

- 同一证据只归入一个主要维度，避免因为同时命中多个 pattern 被重复扣分。
- 每条证据选择一个主要 pattern，编号单独写成 `[S01]`；不要写成 `[S01/C01]`。次要关联可以在解释中说明，但不再次扣分。
- 同类问题在同一段反复出现时按整体强度扣分，不逐词机械累加。
- 先判断保护项 `P01-P03`；被保护的正式语气、局部视角或合理毛边不扣分。
- 总分必须等于各维度得分之和，并与风险区间一致。

按问题强度确定该维度的扣分幅度：

- 轻度：单个词或单句命中，但不影响整体。扣 1-3 分。
- 中度：同类 pattern 多次出现，影响段落可信度。扣 4-8 分。
- 重度：多个 pattern 成簇出现，整段像模板生成。扣 9-15 分。
- 致命：伪造来源、伪造经历、事实改变、高风险建议无来源。直接低于 60 分，并说明原因。

短文本证据不足时仍可评分，但要标注“评分置信度：低 / 中 / 高”。不要用一两句话做过度确定的作者身份判断。

## 评测步骤

1. 先判断文本类型：公众号、营销稿、报告、邮件、技术文档、个人随笔等。
2. 标出 3-8 个最关键的 pattern 命中项。
3. 给出证据句，不要只列词。
4. 对照 95、85、75、65、50 五个锚点，先确定大致区间。
5. 按维度打分，确保分项之和等于总分，同一证据不重复扣分。
6. 给出修改优先级：先事实，再结构，再语言。
7. 短文本补充评分置信度。
8. 对散文、随笔、故事、作文和情感类文本，额外检查是否过度顺滑、过度完整、像被统一润色过的满分作文。
9. 同时检查是否误伤人工痕迹：旁枝叙事、局部视角、方言口气、普通词、必要重复和轻微不均匀不应单独扣分。

```text
维度得分：
- 事实和可信度：xx/25
- 具体性和信息密度：xx/20
- 语言自然度：xx/20
- 结构节奏：xx/15
- 作者声音：xx/10

```

## 误判保护

不要把以下现象单独当成 AI 味：

- 文章很正式。
- 语法完整。
- 使用常见连接词。
- 有标题和列表。
- 缺少第一人称。
- 作者习惯使用短句。
- 段落之间有必要的连贯。
- 正文来自公告、合同、技术手册或新闻通稿。
- 作文、随笔或故事中存在合理旁枝、局部视角、普通词、必要重复、方言口气或轻微笨拙表达。

只有当多个 pattern 聚集，并且造成空泛、模板化、失真或不可信时，才判断 AI 味明显。
