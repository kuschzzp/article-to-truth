# 示例

## 示例 1：营销稿

Before:

```text
在当今快速发展的数字时代，AI 写作工具正在赋能内容创作者，显著提升创作效率，并为未来内容生态带来无限可能。
```

命中 pattern：

- [C02] 宏大背景铺垫
- [L02] 虚浮动词
- [L03] 过度形容
- [C08] 通用乐观结尾

After:

```text
AI 写作工具更适合处理初稿阶段的重复劳动，比如起标题、列提纲、改摘要。它能省掉一部分空转时间，但发布前仍然要补真实案例、来源和作者判断。
```

## 示例 2：公众号技术文

Before:

```text
这不仅是一个代码辅助工具，更是开发者工作方式的一次重要变革。它通过智能化能力提升效率、优化体验、增强协作。
```

命中 pattern：

- [S01] 不仅而且结构
- [C01] 意义膨胀
- [S03] 三件套
- [L04] 名词化堆叠

After:

```text
它能帮开发者写样板代码、补测试草稿、解释陌生函数。真正难的部分还在人工判断：这段代码该不该写，边界条件有没有漏，团队能不能维护。
```

## 示例 3：评测输出

```text
真实感评分：68/100（越高越自然）
AI 味风险：高

命中 pattern：
1. [C02] 宏大背景铺垫：证据句“在数字化浪潮下……”
2. [L02] 虚浮动词：证据句“赋能企业实现增长……”
3. [S03] 三件套：证据句“提升效率、优化体验、创造价值……”

优先修改：
1. 删除开头宏大背景，直接写具体业务问题。
2. 把“赋能、增长”换成工具具体做了什么。
3. 增加适用边界，说明什么团队适合用。
```

## 示例 4：原创生成触发

User:

```text
帮我写一篇关于夕阳的散文诗
```

应触发本技能，读取 `generation.md`、`literary-generation.md`、`literary-reviewer.md`、`patterns.md` 和 `process.md`。第一遍只作内部草稿；子代理可用时执行独立编辑复写，否则走删法终审回退。生成时重点避免：

- [C01] 把夕阳写成宏大人生意义。
- [C08] 用“珍惜当下、未来可期”强行升华。
- [L03] 连续堆叠华丽形容词。
- [S05] 写成海报式伪金句。

处理方向：

```text
从具体画面进入，比如墙面、窗沿、尘埃、人的动作；保留留白，不解释夕阳象征什么。
```

User:

```text
我想写一篇穿越小说
```

应触发本技能，读取 `generation.md`、`literary-generation.md`、`literary-reviewer.md`、`patterns.md` 和 `process.md`。第一遍只作内部草稿；子代理可用时执行独立编辑复写，否则走删法终审回退。生成时重点避免：

- [C02] “在一个神秘而古老的时代”式宏大背景。
- [C01] 主角开场就承担世界命运。
- [S03] 身世、系统、使命三件套堆设定。
- [F02] 用过多小标题拆剧情。

处理方向：

```text
让人物先遇到一个具体麻烦，用动作和场景带出穿越事实，而不是先解释世界观。
```

## 示例 5：情感随笔过顺复审

Before:

```text
友情大概就是这样。它不总是热烈，也不总是贴近。它藏在一把递过来的伞里，藏在多年后见面仍然能自然坐下的一碗面里。人越长大，越知道能被人记得是一件很珍贵的事。
```

命中 pattern：

- [C09] 作文式通用细节
- [L09] 教学解释腔
- [S08] 过度结构化推理
- [S10] 标准温情隐喻

处理方向：

```text
不要继续把文章润得更完整。先删掉负责升华的通用句，保留一两个更具体、更局部的关系痕迹；让段落有一点不对称，不必每段都收束成“友情是什么”。
```

After:

```text
我后来很少说“最好的朋友”。这个词太满，不太像我们。

我更记得一次下雨。她把伞塞给我时没看我，只说：“你别又感冒。”那句话不漂亮，也没什么值得写进作文的道理。但很多年后，我还是会想起她说“又”这个字。
```

## 示例 6：作文类叙事的人工痕迹保护

人工样本常见特征：

- 有具体称呼和局部位置：邻居、奶奶、村口、饭桌、田埂。
- 有旁枝信息：别人议论、家庭关系、一个看似无关的小动作。
- 有不完全解释：作者当时并不知道原因，只是后来才猜到。
- 有普通甚至笨拙的词：不是每句都漂亮，但能看出叙述者站在哪里。
- 有必要重复：主题词可能反复出现，不一定要全部替换成同义词。

需要保护：

```text
我是在饭桌上听奶奶说起这事的。她说隔壁阿婆不太管瓜秧，倒天天喂那几只鸡。那时我也不懂，只觉得她家院子里总有鸡叫，黄狗趴在门口，阿婆剪那盆歪歪的盆栽。
```

不要改成：

```text
这件事充分体现了阿婆独特的生活智慧。她以从容的态度面对劳动，用不同于常人的方式收获了成功，也启发我们在生活中保持自己的风度。
```

原因：

- 第二版命中 [L09] 教学解释腔，把叙事变成主题讲解。
- 第二版命中 [S08] 过度结构化推理，把人物动作都收束成道理。
- 第一版的饭桌、鸡叫、黄狗、盆栽是有效毛边，属于 [P03] 应保护内容。

处理方向：

```text
保留叙述者“听来的、看见的、不太懂的”位置。可以修掉影响理解的硬伤，但不要把旁枝全部清掉，也不要把人物改成只负责证明主题的符号。
```

## 示例 7：文学首次生成的全文复审

初稿可能已经没有套话，却仍然像 AI：停住的钟、凉掉的茶、没寄出的信和窗上的水痕依次出现，每件物品都负责证明“孤独”，每段最后再收成一句漂亮判断。这主要命中 [C10] 和 [S08]，不能靠替换形容词解决。

复审时检查：

- 去掉负责解释主题的金句后，场景能否自己成立。
- 把某件物品换成同类物件后，人物关系和因果是否完全不变。
- 是否所有细节都承担同一种象征功能。
- 物件状态是否连续。例如开头已经把晾晒的衣服收进屋，结尾不能无解释地仍写它挂在原处。
- 指定作家风格时，是否只靠窗纸、煤炉、旧广告等年代道具制造表面相似。

处理方向：

```text
先删去过度解释和重复象征，再把人物真正正在做的事、有限认知和场景因果放回文章。允许自然旁枝存在，但不要机械添加一件“无用小事”作为新的反 AI 模板。完成连续性回查后，只交付终改稿。
```
