# 改写与审稿流程

Article To Truth 的标准流程是“识别 -> 初改 -> 复审 -> 终改”。这个流程用于防止一次改写后仍然保留 AI 痕迹。

## 1. 锁定事实

先标出不能改动的信息：

- 人名、机构名、产品名。
- 时间、地点、数字。
- 来源、引用、法律或合规表述。
- 原文承诺范围。
- 作者明确表达的立场。

没有证据的地方不要补。需要事实时，提示用户补素材或把表达改成限定判断。

## 2. 识别 pattern

从 `patterns.md` 中选出最明显的 pattern。短文本列 1-3 个，长文列 3-8 个。

记录格式：

```text
[C03] 模糊归因：证据句“行业报告显示……”
[L02] 虚浮动词：证据句“赋能创作者……”
```

## 3. 初改

按优先级处理：

1. 事实风险：模糊来源、伪数据、伪个人经验。
2. 内容空泛：意义膨胀、宏大背景、万能价值。
3. 结构模板：三件套、不仅而且、段落同模。
4. 语言润色：机械连接词、虚浮动词、过度形容。
5. 格式清洗：粗体冒号列表、符号装饰、标题复读。

## 4. 复审

初改后问自己：

```text
这版哪里还明显像 AI 写的？
```

至少检查：

- 是否还有空泛价值判断。
- 是否还有模糊归因。
- 是否仍然过度工整。
- 是否把所有句子都改得太顺、太圆。
- 是否为了口语化引入新的模板。
- 是否误删了作者原本的真实痕迹。

## 5. 终改

终稿应该做到：

- 覆盖原文重要信息。
- 读起来有具体对象和动作。
- 保留合适的语气和文体。
- 有边界，不乱承诺。
- 不残留明显机器人口吻。

## 6. 改动说明

用 pattern 编号说明主要改动：

```text
主要改动：
1. [C02] 删除宏大背景，直接进入具体场景。
2. [C03] 去掉无来源的“行业报告显示”。
3. [S03] 拆掉三件套，按真实优先级重排。
```

## 输出策略

如果用户只要“改一下”，输出终稿和简短说明即可。

如果用户要“规则”“评测”“严谨一点”，输出完整 pattern 命中和复审过程。

如果文本很长，先给总体问题和示范改写一段，再询问是否继续逐段处理。
