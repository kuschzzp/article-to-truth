# 改写与审稿流程

Article To Truth 的标准流程是“识别 -> 初改 -> 复审 -> 终改”。这个流程用于防止一次改写后仍然保留 AI 痕迹。

## 1. 锁定事实

先标出不能改动的信息：

- 人名、机构名、产品名。
- 时间、地点、数字。
- 来源、引用、法律或合规表述。
- 原文承诺范围。
- 作者明确表达的立场。
- 否定、例外和条件词，例如“不受影响”“仅限”“最多”“除非”“无需”。

没有证据的地方不要补。需要事实时，提示用户补素材或把表达改成限定判断。

改写前在内部建立事实锁定表：

| 类型 | 原文内容 | 终稿要求 |
|---|---|---|
| 主体与对象 | 谁、什么产品、哪个接口 | 主体和指向不变 |
| 数字与日期 | 数值、单位、时间窗口 | 数值、单位和时间关系不变 |
| 范围与限制 | 最多、仅限、适用于、不包含 | 不扩大、不缩小 |
| 否定与例外 | 不受影响、无需、除外 | 否定关系和例外对象不变 |
| 来源与立场 | 谁说的、作者判断、引用 | 不把判断升级为事实 |

终改后逐项回查。新增数字、来源、案例、因果关系或承诺都视为事实漂移；无法从原文推出的内容应删除或明确标成有限建议。

## 2. 识别 pattern

从 `patterns.md` 中选出最明显的 pattern。短文本列 1-3 个，长文列 3-8 个。

记录格式：

```text
[C03] 模糊归因：证据句“行业报告显示……”
[L02] 虚浮动词：证据句“赋能创作者……”
```

额外判断文本是否属于散文、故事、作文、个人随笔或情感文。如果属于，先看它是否被润得过于干净：

- 叙述视角是否太全知，像作者什么都知道。
- 段落是否每一段都刚好完成一个论点。
- 细节是否平均分配，全部服务主题，没有旁枝。
- 语言是否太中性、太漂亮，缺少作者偏心或临场选择。
- 句式是否反复使用同一种递进、对照或升华。
- 核心行动已经能表达主题后，是否仍追加同向物件、声音和感受。
- 是否用抽象旁白或“好像、仿佛、倒像”一类比喻替现场标注情绪。
- 是否出现不推动事件的语义巧合，或把标题物件在结尾处理得过于圆满。

## 3. 初改

按优先级处理：

1. 事实风险：模糊来源、伪数据、伪个人经验。
2. 内容空泛：意义膨胀、宏大背景、万能价值。
3. 结构模板：三件套、不仅而且、段落同模。
4. 过顺痕迹：过度结构化、低跳跃度、标准温情隐喻、作文式通用细节、无旁枝叙事。
5. 精修痕迹：细节平均分配、主题证据重复、语义巧合、情绪标注式比喻、过度中性、精修无毛边、可预测句法。
6. 语言润色：机械连接词、虚浮动词、过度形容、教学解释腔、词汇过度统一。
7. 格式清洗：粗体冒号列表、符号装饰、标题复读。

改写原则：

- 先删掉通用升华和模板结构，再处理词句漂亮不漂亮。
- 能保留原文已有毛边时，不要把它改成更顺的“标准答案”。
- 可以保留一点普通词、重复、短停顿和不完全解释；不要故意写错或制造混乱。
- 没有素材时，不要凭空加入人名、地点、方言、家庭关系或亲身经历。

## 4. 复审

初改后问自己：

```text
这版哪里还明显像 AI 写的？
```

至少检查：

- 是否还有空泛价值判断。
- 是否还有模糊归因。
- 是否仍然过度工整。
- 是否把所有句子都改得太顺、太圆。
- 是否每段都像在完成一个标准论点。
- 是否所有细节都刚好服务主题，反而像作文素材。
- 是否用了路灯、外套、雨伞、一碗面这类通用温情隐喻。
- 是否把随笔、散文或故事写成了概念解释。
- 核心行动已经足够后，是否仍用抽象旁白、情绪比喻或新道具重复解释主题。
- 是否有一句概括能被直接删掉，而前后动作与信息完全不受影响；如是，优先删除。
- 是否有“恰好”呼应主题的字词、光线、广播、物件位置等语义巧合。
- 是否过度中性，读不出作者的偏心、疑惑、迟疑或情绪温差。
- 是否细节分布太均衡，每段都像按大纲摆证据。
- 是否句式太可预测，反复用对照、递进和收束句。
- 是否删掉了原文里有效的旁枝、普通词、必要重复或局部视角。
- 是否为了口语化引入新的模板。
- 是否误删了作者原本的真实痕迹。
- 人物方向、物件状态、数量与容器是否连续，楼上楼下、门内门外等相对位置是否写反。
- 结尾是否把标题或核心物件修好、放稳、熄灭、归位得过于完整，形成预定式闭环；暂时删掉末尾一句再比较。

## 5. 终改

终稿应该做到：

- 覆盖原文重要信息。
- 读起来有具体对象和动作。
- 保留合适的语气和文体。
- 保留一点合理的不均匀：段落长短、判断出现位置和细节解释程度不必完全对称。
- 对随笔、故事和作文类文本，保留合理的叙述位置：作者当时看见什么、听见什么、不知道什么。
- 对文学文本，核心行动承担主题，旁白不再复述；比喻只有在增加观察信息时才保留。
- 文学结尾停在人物仍在进行的动作、现场余留或有限判断上，不为标题物件制造标准答案式闭环。
- 有边界，不乱承诺。
- 不残留明显机器人口吻。
- 数字、日期、单位、否定、例外和条件关系与事实锁定表一致。

## 6. 文学删法终审

散文、随笔、散文诗、小说、故事和情感文本完成终改后，必须再做一次独立的删法终审。此时停止创作，只做编辑：

1. 从头到尾逐句读终改稿，先删或字面化不增加观察信息的物件拟人、情绪比喻和漂亮类比。
2. 检查概括“人、生活、许多事情、屋子原来如何”的旁白；整句删除后人物行动、事实和因果不受影响，就直接删除。
3. 删除核心行动已经传达主题后继续出现的同向声音、远景、旧物、动物、文字残片和感受句。
4. 暂时删掉末尾一句；若标题物件不再被过分圆满地修好、放稳、熄灭或归位，且事件仍完整，就采用删后的结尾。
5. 删除后只修复必要的语法、指代和衔接，不得再添加新比喻、新细节、新象征、新金句或新的主题解释。

删法终审不是把文章削成流水账。推动事件的动作、暴露人物局限的观察、改变关系的对话和维持因果所需的细节应保留。比较终改稿与删后稿，只要删后稿没有损失用户要求、核心事件和必要连续性，就交付删后稿。

## 7. 改动说明

用 pattern 编号说明主要改动：

```text
主要改动：
1. [C02] 删除宏大背景，直接进入具体场景。
2. [C03] 去掉无来源的“行业报告显示”。
3. [S03] 拆掉三件套，按真实优先级重排。
```

## 输出策略

识别、初改和复审默认是内部过程，不需要逐步展示。

只改写时使用：

```text
终稿：
...

主要改动：
1. [C01] ...
2. [L03] ...
```

用户明确说“只输出终稿”或“只要正文”时，省略改动说明和所有内部过程；上下文已清楚时也可以省略“终稿：”标签。

评测后再改写时，按这个顺序输出：

```text
原文真实感评分：xx/100（越高越自然）
原文 AI 味风险：低 / 较低 / 中 / 高 / 很高
主要命中 pattern：...（附证据句）

终稿：
...

终稿真实感评分：xx/100（越高越自然）
终稿 AI 味风险：低 / 较低 / 中 / 高 / 很高
剩余风险：...
```

只改写时不扩展为评分；只评测时不输出完整终稿。不要再输出方向相反的“AI 味评分”或生成概率。

如果用户要“规则”“评测”“严谨一点”或明确要求展示过程，再输出完整 pattern 命中和复审过程。

如果文本很长，先给总体问题和示范改写一段，再询问是否继续逐段处理。
